# Clash 配置使用说明（OpenClash / Clash Verge / Clash Meta）

该仓库包含三份常用的 Clash 配置/覆写模块：

- `clashverge全局扩展覆写配置dns防泄漏自动去除广告.txt` — 针对 Clash Verge（Android）/Clash 客户端的全局覆写与广告拦截模块。
- `openclash覆写模块dns防泄漏去广告.txt` — 针对 OpenClash（OpenWrt） 的覆写模块，包含 DNS 防泄漏 与 广告去除规则。
- `多订阅合并模板dns防泄漏自动去除广告.yaml` — 多订阅合并模板，便于将多个订阅链接合并成一个配置文件，并包含 DNS 防泄漏与广告规则。

下面为每一份配置提供详细的使用指南、常见问题与建议：

---

## 1) clashverge全局扩展覆写配置dns防泄漏自动去除广告.txt

目的
- 为在 Android 上运行的 Clash Verge / Clash 类客户端提供一份“全局覆写”模块，自动去广告并减少 DNS 泄漏风险。

使用场景
- 你已在手机上安装 Clash Verge、Clash for Android、Clash Meta 等，并希望在现有订阅/配置基础上，额外应用统一的广告拦截、DNS 防泄漏和一些全局覆写规则。

使用步骤
1. 下载本文件到手机或通过 USB/网盘传到手机。建议把文件改名为 `clashverge-覆盖.yml` （把后缀改为 .yaml/.yml 更易被客户端识别）。
2. 打开 Clash Verge 或 Clash 客户端：
   - 进入 Profiles（配置）或配置导入页面。选择“从文件导入”或“Import from file”。
   - 选择刚才保存的 YAML 文件并导入。
3. 导入后：
   - 在 Profiles 中启用该配置（或在主配置内启用覆写/扩展模块，取决于客户端如何支持覆写）。
   - 如果你是把这份文件作为“覆写模块”单独导入，请确保主配置没有与本覆写相冲突的同名 section（如重复的 proxy-groups 名称等）。

注意事项
- 如果客户端不接受 .txt 后缀，请改为 .yaml 或 .yml。
- 覆写模块通常会覆盖主配置中的同名字段，导入前请做好备份。若需回退，重新切回原始订阅配置即可。

常见问题
- 导入失败或报错：检查文件是否包含 BOM 或 Windows CRLF，建议用 UTF-8 无 BOM 保存并上传。
- DNS 泄漏仍然存在：确认主配置或客户端 DNS 模式已设为 fake-ip 或 redir-host（取决于客户端支持），并确保本覆写的 dns 节点没有被主配置覆盖。

## 2) openclash覆写模块dns防泄漏去广告.txt

目的
- 为 OpenClash（OpenWrt） 提供一份易用的覆写模块，包含广告过滤规则和 DNS 防泄漏设置，适用于路由器端统一防护。

使用场景
- 你在基于 OpenWrt 的路由器上运行 OpenClash，希望通过覆写模块增强广告拦截并避免 DNS 泄漏。

使用步骤
1. 下载并保存本文件到你的电脑。建议重命名为 `openclash-overwrite.yaml`（使扩展名为 .yaml）。
2. 通过 LuCI（OpenWrt 的网页管理界面）：
   - 登录路由器管理页面：Services > OpenClash（或插件所在位置）。
   - 找到配置管理或“覆写配置 / 自定义配置”区域（OpenClash 的版本不同，位置会有差异）。
   - 将文件内容作为覆盖/覆写粘贴到自定义配置框，或上传并保存为一个新的配置文件。
3. 保存并应用配置，等待 OpenClash 重新加载并生效。

建议设置（路由器端）
- 确保 OpenClash 的 DNS 模式设置为 fake-ip 或 redir-host，以配合本配置的防泄漏设置。
- 在 OpenClash 的“常规设置”中启用“启用 DNS 代理/本地 DNS”，并填写可信赖的上游 DNS（例如 223.5.5.5、1.1.1.1、8.8.8.8 等）。

注意事项
- 覆写模块可能会覆盖主配置的某些字段，请在修改前备份原始配置文件（OpenClash 通常会保留历史配置快照）。
- 如果你使用了额外的 rule-provider 或者自定义 provider，请确保命名不冲突。

常见问题与排查
- 应用后网络异常：回退到备份配置，逐项检查本覆写中可能影响路由或 proxy-groups 的规则。
- 广告仍未拦截：目标网站可能使用了新的域名或 CDN，需更新拦截列表或使用额外的规则源。

## 3) 多订阅合并模板dns防泄漏自动去除广告.yaml

目的
- 提供一份多订阅合并的模板，方便把多个订阅（不同供应商或不同配置）合并为一份统一的本地配置，同时包含 DNS 防泄漏设置与广告拦截规则。

使用场景
- 你有多个订阅链接（例如不同的订阅提供商、分组或地域节点），希望将它们合并并统一管理。

使用步骤
1. 下载 `多订阅合并模板dns防泄漏自动去除广告.yaml` 到本地。
2. 用文本编辑器（推荐 VS Code、Notepad++、或手机上的文本编辑器）打开文件，找到标记为 `# ======= 订阅替换区 =======` 的区域（或文件内明显的 placeholder）。
3. 将你的订阅 URL 按说明替换或添加到模板里。例如：

   - 如果模板使用占位字段，请把 `YOUR_SUB_URL_1`、`YOUR_SUB_URL_2` 替换为真实的订阅链接。
   - 如果模板包含 `proxies:` 或者 `proxy-provider` 的引用，请保持模板格式不变，仅替换链接或 provider 的 urls 列表。

4. 保存文件（确保为 UTF-8 格式且扩展名为 .yaml 或 .yml）。
5. 导入到你使用的客户端：
   - OpenClash：通过 LuCI 上传并应用为一个配置（或将其作为合并后的本地配置）。
   - Clash Verge / Clash Meta（Android）：将文件传到手机并在 Profiles 中“从文件导入”。

合并策略说明（通用）
- 模板将多个订阅的节点合并到统一的 proxies 集合，并配置了若干常用的 proxy-groups（如自动选择、代理直连、阻断等）。
- 模板自带广告拦截与 DNS 防泄漏设置（dns 部分），建议保留这些设置以获得更好隐私与广告拦截效果。

注意事项
- 合并后的配置文件体积可能较大（节点较多），某些客户端在解析时可能略慢，尤其是在手机端。
- 如果需要按地域或节点标签分组，建议编辑模板内的 proxy-groups 部分，按照你的偏好调整分组策略。

常见问题
- 节点不可用或延迟高：可能是订阅源中的节点已失效或网络不可达，尝试刷新订阅或移除不稳定节点。
- 导入时发生语法错误：YAML 对缩进非常敏感，编辑时请确保缩进正确并使用空格（不要用 Tab）。

## 常规建议与维护

- 备份：在替换或导入任何新配置之前，先备份现有工作配置（无论是 OpenClash 的历史快照，还是客户端的导出文件）。
- 更新：广告规则与域名会随时间变化，建议定期（例如每周或每两周）更新你的配置或订阅。
- DNS：要最大限度减少 DNS 泄漏，客户端/路由器应使用模板中的 dns 设置（例如 enhanced-mode/fake-ip、并指定可信上游 DNS）。
- 调试：大多数客户端都会在日志中给出导入/生效错误信息，遇到问题请先查看日志，以便快速定位。
